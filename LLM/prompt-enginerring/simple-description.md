# 提示工程 [Prompt Engineering]

## 基本概念

### 什么是提示工程

Prompt Engineering（提示词工程），简单来说，就是设计、撰写和优化输入给人工智能（尤其是大语言模型，如 GPT、Gemini、Claude 等）的文本（即 Prompt），以引导 AI 准确、高效地输出你期望的结果的这一过程。

如果把大语言模型比作一个无所不知但缺乏上下文的“超级大脑”，那么提示词工程就是连接人类意图与这个大脑的“翻译器”和“控制台”。

### 提示工程核心组成部分

虽然一个简单的提问（如“什么是光合作用？”）也是 Prompt，但专业的提示词工程通常包含以下一个或多个核心要素：

- 角色/设定(Rile/Context)：告诉AI它是谁。例如：“你是一位资深的软件架构师...”
- 任务/指令(Instruction)：明确要求AI做什么。例如：“请重构以下前端代码，提高其可读性...”
- 上下文/背景(Background/Data)：提供必要的背景信息或原始数据。
- 输入数据(Input Data)：需要处理的具体内容。
- 控制/限制条件(Constraints)：限制输出的边界。例如：“字数控制在 200 字以内”、“只能返回 JSON 格式”、“不要解释原因”。
- 示例(Example)：给出“输入-输出”的样例（即Few-shot提示法），让AI模仿。

### 常见的提示词工程术语

随着技术的发展，提示词工程已经从单纯的“字斟句酌”演变为一套结构化的技术方法：

#### 1. 零样本提示(Zero-shot Prompting)

不对模型进行任何示例训练，直接下达指令。依赖于模型本身的强大泛化能力。

示例： “将以下句子翻译成英文：‘今天天气真好。’”

#### 2. 少样本提示(Few-shot Prompting)

在提供最终指令前，先给模型提供一个或多个期望输出的示例。这对于统一输出格式、复杂逻辑任务或特定的代码/文本风格非常有效。

#### 3. 思维链提示(Chain-of-Thought, Cot)

引导模型在输出最终答案之前，一步一步写出推理过程。这能显著提升模型在数学、逻辑推理和复杂决策任务中的准确率。

示例： “请一步步思考并解决这个问题：小明有5个苹果，吃了2个，又买了一个3苹果的袋子，现在他有几个苹果？”

#### 4. 自动化与Agent技能封装(Tool/Skill Prompting)

在更高级的应用中（如 AI Agent/智能体开发），提示词工程用于定义 AI 的“技能边界”和“工具调用逻辑”。通过系统提示词（System Prompt）严格规范 AI 何时调用 API、如何处理异常流程，确保其在自动化工作流中稳定、可靠地运行，不绕弯路、不走偏。

### 为什么提示词工程如此重要

1. 最大化挖掘模型潜力： 同样的模型，优秀的 Prompt 和糟糕的 Prompt 得到的结果天差地别。
2. 降低开发成本： 在很多场景下，通过优化 Prompt 就能达到理想效果，无需花费高昂的成本去微调（Fine-tuning）模型。
3. 确定性与稳定性： 在企业级应用和自动化脚本中，提示词工程能够对 AI 的行为进行“控域”，使其输出符合预期的、结构化的（如 JSON）稳定结果。

简而言之，提示词工程不仅是一门科学（需要理解模型的底层逻辑与边界），也是一门艺术（需要精准、清晰的语言表达能力）。它是目前人类与 AI 高效协同的核心桥梁。
