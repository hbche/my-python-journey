## 📚 模块一：机器学习经典算法——**用代码学会，公式后补**（3周）

### 核心策略

**不要在理论上纠结**。每个算法先看一个直观视频（5分钟），然后直接看代码实现，跑通，修改参数观察效果。最后才看公式推导。

### 第1周：k-近邻（kNN）—— 最简单的“记忆”算法

- **学习目标**：理解距离度量、决策规则。
- **动手项目**：手写数字识别（MNIST子集），用kNN分类。
- **代码实现**：自己写一个`class KNN`，包含`fit`（只存储数据）和`predict`（计算距离并投票）。
- **理论补丁**：只需知道欧氏距离公式、多数投票原则。暂时不管KD树。
- **资源**：
  - 视频：StatQuest的“KNN clearly explained”（10分钟）
  - 代码参考：《机器学习实战》第2章（你已有这本书，正好用上）

### 第2周：线性回归 —— 理解“学习”的本质

- **学习目标**：理解损失函数（MSE）和梯度下降的直觉。
- **动手项目**：
  1. 用`sklearn`的线性回归拟合房价数据（波士顿房价）。
  2. **手写实现**：用numpy实现批量梯度下降，画出损失下降曲线。
- **关键点**：推导公式可跳过，但必须手动计算一次梯度（对单个样本求导）。
- **资源**：
  - 视频：3Blue1Brown的“梯度下降法”（可视化极佳）
  - 代码：自己写，不要复制。可以用`np.linalg.lstsq`对比结果。

### 第3周：逻辑回归 —— 进入分类世界

- **学习目标**：理解Sigmoid函数、交叉熵损失。
- **动手项目**：
  1. 用逻辑回归做垃圾邮件分类（可用简单特征）。
  2. 手写实现：从零实现逻辑回归的训练和预测。
- **理论补丁**：只需知道Sigmoid将输出压缩到0~1，交叉熵比MSE更适合分类。
- **资源**：
  - 视频：Andrew Ng的“Logistic Regression”章节（Coursera旧版可看）
  - 注意：你手头的《机器学习实战》第5章就是逻辑回归，直接跟着敲。

### 模块一检验

- 能用自己的kNN、线性回归、逻辑回归代码，在小型数据集上得到合理结果。
- 能解释梯度下降的直观目的（下山找最低点）。

### 补充说明

#### 为什么第一层只选这三个？

我们的目标是**理解Transformer**，它本质上是一个**深度神经网络** + **自注意力机制**。为了理解神经网络，你需要：

| 所需前置知识         | KNN | 线性回归 | 逻辑回归 |
| -------------------- | --- | -------- | -------- |
| 距离度量 / 相似度    | ✅  |          |          |
| 损失函数（MSE）      |     | ✅       |          |
| 梯度下降（参数更新） |     | ✅       |          |
| 激活函数（Sigmoid）  |     |          | ✅       |
| 分类概率输出         |     |          | ✅       |
| 过拟合 / 欠拟合      |     | ✅       | ✅       |

这三个算法刚好为你搭建了从“传统机器学习”到“神经网络”的桥梁。而**其他经典算法（决策树、SVM、聚类、降维等）** 在理解大模型的过程中**不是必需的**，它们属于另一个知识分支（传统ML工程场景）。

---

#### 但是你可能会问：“那我岂不是漏掉了半壁江山？”

你说得对。如果目标是“全面掌握大模型相关技能”，那么第一层的确是**精简过**的。在后续的层（比如LLMOps、Agent、RAG）中，你完全不需要决策树或SVM。但是，如果你将来要做**数据分析、特征工程、传统评分卡、异常检测**等任务，那些算法就有用了。

**所以，我的建议是**：

> 先拿下这三个，完成第一层主线任务。等你能够熟练训练神经网络、理解Transformer之后，再花**2周时间**回头补**决策树/随机森林**和**K-Means聚类**。这两个最常用，且对直觉有帮助，但不需要在第一层就学。

## 🧠 模块二：神经网络入门——**从感知机到反向传播**（2周）

### 第4周：感知机与多层网络

- **学习目标**：理解神经元、激活函数、层。
- **动手项目**：用numpy实现一个单层感知机，解决AND门问题。
- **关键突破**：理解为什么单层感知机不能解决XOR问题 → 引出多层。

### 第5周：反向传播 —— 不背公式，手算一次

- **学习目标**：直观理解链式法则如何传递误差。
- **动手项目**：
  1. 手算一个2层网络（2输入→2隐藏→1输出）的一次前向和反向传播，用Excel或纸上演算。
  2. 用numpy实现这个2层网络，训练解决XOR问题。
- **资源**：
  - 视频：3Blue1Brown的“反向传播”系列（强烈推荐）
  - 代码参考：Andrej Karpathy的“Micrograd” —— 一个100行Python实现的自动微分引擎，非常透彻。

### 模块二检验

- 你能不参考代码，画出一个小型神经网络的前向和反向传播数据流图。
- 你写的numpy神经网络能够收敛。

## 🔥 模块三：PyTorch深度学习框架（3周）

现在开始使用现代工具，放弃手写numpy梯度。

### 第6周：PyTorch基础

- **学习目标**：Tensor操作、自动梯度（autograd）、优化器。
- **动手项目**：用PyTorch重写之前的逻辑回归和2层神经网络。
- **资源**：PyTorch官方教程“60分钟入门”。

### 第7周：卷积神经网络（CNN）

- **学习目标**：理解卷积、池化、感受野。
- **动手项目**：用PyTorch搭建一个简单的CNN（2个卷积层+全连接），在MNIST上达到>98%准确率。
- **理论补丁**：不需要推导卷积公式，只需理解感受野和参数共享。

### 第8周：训练技巧与调参

- **学习目标**：过拟合、正则化、Dropout、学习率调度。
- **动手项目**：在CIFAR-10上训练一个CNN，尝试加入数据增强和Dropout，观察验证集准确率变化。
- **资源**：Fast.ai的课程（实战导向）。

### 模块三检验

- 能用PyTorch在10分钟内训练一个图像分类模型。
- 知道何时使用`model.train()`和`model.eval()`。

## 🚀 模块四：Transformer核心原理——**大模型的基石**（3周）

这是第一层的终极目标。你需要**用代码理解**，而不是死记公式。

### 第9周：自注意力机制（Self-Attention）

- **学习目标**：理解Query、Key、Value的计算过程。
- **动手项目**：用numpy实现一个简单的自注意力层（单头，无缩放）。输入一个小句子（3个词，每个词向量维度4），手工计算注意力权重。
- **资源**：
  - 视频：The Illustrated Transformer（博客文章 + 动画）
  - 代码：参照“numpy实现的Transformer”教程。

### 第10周：Transformer Block（多头注意力 + 前馈网络 + 残差连接）

- **学习目标**：组装一个完整的Encoder层。
- **动手项目**：用PyTorch实现一个`TransformerBlock`类，包含多头注意力（2个头）、LayerNorm、残差连接。
- **理论补丁**：不用深究位置编码的公式，只需要知道“为什么需要位置信息”。

### 第11周：从零实现一个迷你GPT（字符级）

- **学习目标**：理解语言模型的训练和生成过程。
- **动手项目**：
  - 使用Andrej Karpathy的`minGPT`或`nanoGPT`，但要求自己敲一遍核心代码。
  - 在一个小文本文件（如莎士比亚作品片段）上训练一个字符级GPT，让它生成新的文本。
- **关键理解**：训练时如何计算交叉熵损失？生成时如何采样？

### 模块四检验

- 你能够画出Transformer Encoder的结构图（带数据流的形状）。
- 你训练的小GPT能够生成语法基本正确的英文短句。

## 🎯 核心建议：如何克服“枯燥、忘记、坚持不下去”

1. **代码先行，理论后补**：每个新算法，先复制一份能跑的代码，改了看效果，再回头理解。
2. **每周必须有一个“能跑通”的项目**：哪怕只是打印出损失值下降，也比读三章书有用。
3. **使用Jupyter Notebook**：把代码、注释、可视化放在一起，方便复习。
4. **建立自己的“公式卡片”**：只记录最核心的3~5个公式（如MSE、Sigmoid、交叉熵），贴桌面上。
5. **加入学习社群**：找一个AI学习群（如Datawhale），每周打卡项目。
