# 第 1 章 机器学习概览

光学字符识别（Optical Character Recognition，OCR）属于机器学习的一个应用。

上世纪90是年代垃圾邮件过滤器是第一个风靡全球的机器学习应用。

## 1.1 什么是机器学习

一个计算机程序利用经验E来学习任务T，性能是P，如果针对任务T的性能P随着经验E不断增长，则称为机器学习。

## 1.2 为什么使用机器学习

机器学习擅长处理的场景：

1. 现有解决方案需要有大量微调或一长串规则来解决的问题。
2. 使用传统方法无法解决的复杂问题
3. 变化的环境（机器学习系统可以很容易地根据新数据重新训练数据，始终保持最新状态）
4. 深入了解复杂问题和大量数据

## 1.3 应用示例

1. 分析产品线上的产品图像来对产品进行分类。这是图像分类，通常使用卷积神经网络（CNN），有时使用Transformer
2. 通过脑部扫描发现肿瘤。这是语义图像分割，其中图像中的每个像素都被分类（我们想确定肿瘤的确切位置和形状），通常使用CNN或Transformer。
3. 自动分类新闻文章。这是自然语言处理，更具体的说是文本分类，可以使用循环神经网络和CNN来解决，但Transformer效果更好。
4. 自动标记论坛中的恶评。这也是文本分类，使用相同的NLP工具。
5. 自动对长文档做总结。这是NLP的一个分支，称为文本摘要，使用相同的工具。
6. 创建聊天机器人或个人助理。这涉及许多NLP组件，包括自然语言理解（Natural Language Understanding，NLU）和问答模块。
7. 根据许多绩效指标来预测公司下一年的收入。这是一个回归任务（即预测值），可以使用任何回归模型来解决。
8. 让应用程序对语音命令做出反应。这是语音识别，需要处理音频样本，通常使用RNN、CNN或Transformer进行处理。
9. 检测信用卡欺诈。这是异常检测，可以使用隔离森林、高斯混合模型或自动编码器来解决。
10. 根据客户的购买情况对客户进行细分，从而可以为每个细分涉及不同的营销策略。这就是聚类，可以使用K均值、DBSCAN等来实现。
11. 在清晰又有洞察力的图表中表示复杂的高维数据集。这是数据可视化，通常涉及姜维技术。
12. 根据过去的购买情况推荐客户可能感兴趣的产品。这是一个推荐系统。通常使用人工神经网络来处理。
13. 为游戏构建智能机器人。这通常使用强化学习（Reinforcement Learning，RL）来解决。

## 1.4 机器学习系统的类型

- 在训练期间是否受到监督（监督、无监督、半监督、自我监督等）
- 是否可以即时增量地学习（在线学习、批量学习）
- 是通过简单地将新数据与已知数据进行比较来工作，还是通过检测训练数据中的模式并建立预测模型来工作（基于实例的学习与基于模型的学习）

### 1.4.2 批量学习与在线学习

是否从输入数据流中进行增量学习来分为在线学习和批量学习（或称为离线学习）。

| 类型     | 说明                                                                                 | 特点                                                                                                                                                               |
| -------- | ------------------------------------------------------------------------------------ | ------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| 在线学习 | 通过以单独的数据或小批量的小组数据方式循环地向系统提供数据实例来对系统进行增量训练。 | 每个学习步骤既快速又便宜。如果学习率很好，会很快适应新数据，但是也会很快忘记旧数据。如果学习率很低，系统会有更多的惰性，学习慢，但它对新数据中的噪声敏感度也会降低 |
| 批量学习 | 必须使用所有可用的数据进行训练。随着时间推移，模型不能很好的拟合新数据。             | 又叫离线学习。存在模型腐烂或数据漂移。                                                                                                                             |

## 1.5 机器学习的主要挑战

### 1.5.1 训练数据量不足

大多数机器学习算法需要大量数据才能正常工作。

### 1.5.2 训练数据不具代表性

也叫采样偏差。为了很好地实现泛化，训练数据必须能代表我们想要泛化的新样例。

### 1.5.3 低质量数据

如果我们的训练数据充满错误、异常值和噪声，系统将更难检测到底层模式，也就更不太可能表现良好。花时间清洗训练数据通常是非常值得的。

### 1.5.4 无关特征

机器学习项目成功的一个关键部分是提取出好的特征集来进行训练。这个过程叫特征工程，包括特征选择、特征提取、创建特征集。
